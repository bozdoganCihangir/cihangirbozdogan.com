# Claude Context — cihangirbozdogan.com

## What this is

A personal, zero-cost static news site for **Cihangir Bozdogan**. Daily curated tech & AI signal — trending tools/models/APIs, news from a few high-signal sources, and a curated roster of practitioner blogs.

There is **no database**, **no auth**, **no API**, **no users other than me**. Content is generated by a slash command and committed to git. A GitHub Actions workflow auto-deploys to GitHub Pages on push to `main`.

## How it works

1. `/refresh` (slash command, defined in `.claude/commands/refresh.md`) runs all the fetching, ranking, and summarization.
2. It reads `lib/sources.ts` — **the single source of truth** for feeds, lookback windows, item caps, and the curated author roster.
3. It **fully overwrites** `content/news.json` with a fresh `NewsPayload` (schema in `lib/types.ts`).
4. It commits and pushes to `main`. GitHub Actions builds the static export and redeploys to GitHub Pages in ~1min.

There is no history. Each refresh is a snapshot.

## Pages

| Route | Content | Sidebar |
|-------|---------|---------|
| `/` | News sections — Hacker News, Reddit, GitHub Trending, Blogs & Newsletters | Anchor TOC (sections) |
| `/voices` | Curated practitioner blogs — latest posts (last 30 days, max 5 per author) | Anchor TOC (authors) |
| `/trending` | Full-detail view of trending tools / models / APIs / resources | Anchor TOC (categories) |

Top nav (`components/nav.tsx`) is sticky on every page. Layout shell is `components/page-shell.tsx`.

## Tech Stack

- **Next.js 16** (App Router, Turbopack) — all pages prerendered as static
- **React 19**, **TypeScript**
- **Tailwind CSS v4** (theme tokens in `app/globals.css`)
- **Inter** + **Source Serif 4** via `next/font`
- **GitHub Pages** for hosting (static export via `output: "export"`, deployed by GitHub Actions)
- **No DB, no backend, no API routes**

## Editing rules

- **Adding a feed** (HN/Reddit/GitHub/RSS) → edit `lib/sources.ts` only. The `/refresh` command reads it; no other code changes needed.
- **Adding/removing a Voices author** → edit the `voices.authors[]` array in `lib/sources.ts`. The display order in the array is the display order on the page — keep it intentional.
- **Changing display layout** → components only. The data shape lives in `lib/types.ts`; if you change the shape, update `/refresh` to match.
- **Adding a category** (currently only `tech`) → add a new `CategoryConfig` in `lib/sources.ts`, then call `/refresh <category>`.

## SEO

- **Single source of truth**: `lib/seo.ts` (site URL, name, tagline, description, keywords, author location/title). Change the domain or tagline there — every page, OG image, sitemap, and JSON-LD picks it up.
- Per-page `Metadata` lives in each route's `page.tsx` and uses the `%s · Cihangir Bozdogan` title template from `app/layout.tsx`.
- `Person` + `WebSite` JSON-LD is injected in `app/layout.tsx`. To add `sameAs` (GitHub/LinkedIn/X), append URLs to the `personLd` object — this is the highest-leverage SEO signal for Google's Knowledge Graph.
- Search Console verification: paste the token into the `verification.google` field in `app/layout.tsx` (currently commented out) and push.
- Favicon and Apple touch icon are static PNGs (`app/icon.png`, `app/apple-icon.png`). The OG/Twitter card is a static image at `public/og.png`, referenced everywhere via the `OG_IMAGE` constant in `lib/seo.ts`. It is **not** generated by `next/og` anymore — GitHub Pages types files by extension, so an extensionless dynamic route would ship as `application/octet-stream` and break social previews. **Regenerate `public/og.png` by hand if the branding, name, or tagline changes.**

## Style

- Editorial / FT-inspired pink-paper palette. Tokens in `app/globals.css`.
- Source Serif 4 for headlines/body prose; Inter for UI chrome.
- Collapsible items (news + trending) use a CSS-grid `grid-template-rows` animation, not `max-height`.
- Voices items are **not** collapsible — title + summary visible by default. No avatars, no dates, no metadata. Text + link only.

## Rules of engagement

- **No tests.** No unit, no e2e, no playwright. Don't add them.
- **No docs unless asked.** This file and the `/refresh` command doc are the only docs.
- **No new dependencies without asking.** This site is deliberately tiny.
- **`content/news.json` is generated.** Don't hand-edit it (except for one-off fixes the user explicitly asks for).
- **Don't fabricate data.** If a feed fails during `/refresh`, report the failure and ship without those items. Never invent posts to fill gaps.
- **Don't push without asking** unless the work was explicitly a `/refresh` run (which always commits + pushes — that's its job).
- **Build must pass.** Run `pnpm build` before declaring work done — it's the only verification we have.

## Deployment

- Auto-deploy: push to `main` → GitHub Actions (`.github/workflows/deploy.yml`) runs `next build` (static export → `./out`) → deploys to GitHub Pages → live in ~1min.
- Repo **Settings → Pages → Source** must be set to **GitHub Actions** (not "Deploy from a branch").
- Custom domain `www.cihangirbozdogan.com` is pinned by `public/CNAME`; DNS must point at GitHub Pages (`www` CNAME → `bozdogancihangir.github.io`, apex A records → GitHub Pages IPs).
- No env vars needed. No secrets. No `.env` file.
- Everything is public, everything is static.
